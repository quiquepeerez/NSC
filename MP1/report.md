# Mandelbrot Set – Performance Scaling Report

---

## 1. The Problem

The Mandelbrot set is a well-known fractal defined in the complex plane.  
For each complex number

$$
c = x + iy
$$

we compute the iterative sequence:
$$
z_(n+1) = z_n² + c
z_0 = 0
$$


A point belongs to the Mandelbrot set if the magnitude of `z` remains bounded.  
In practice we use an escape condition:
$$
|z| > 2
$$


If this condition occurs, the point is considered to have escaped. Otherwise, if the sequence does not escape after a fixed number of iterations (`max_iter`), the point is assumed to belong to the set.

The Mandelbrot image is generated by evaluating this iterative process for every pixel in a grid covering a region of the complex plane.

Because each pixel requires multiple iterations, computing large grids becomes computationally expensive. This makes the Mandelbrot set a useful benchmark for comparing different numerical computing strategies in Python.

**Configuration used in the experiments**

| Parameter | Value |
|----------|------|
| Domain | x ∈ [-2.0, 1.0], y ∈ [-1.5, 1.5] |
| Max iterations | 100 |
| Grid sizes tested | several increasing grid sizes |

## 2. Implementations

Three implementations were developed to compute the Mandelbrot set. Each method illustrates a different approach to numerical computation in Python.

---

### 2.1 Naive Implementation

The naive implementation uses nested Python loops to compute each pixel independently.

```python
def mandelbrot_naive(width, height, max_iter):
    result = np.zeros((height, width), dtype=np.float64) #Create a 2D array to hold the escape values

    for row in range(height):
        for col in range(width):
            # Map pixel -> complex plane
            x = X_MIN + (col / (width - 1)) * (X_MAX - X_MIN) #Map pixel column to real part
            y = Y_MIN + (row / (height - 1)) * (Y_MAX - Y_MIN) #Map pixel row to imaginary part

            c = complex(x, y)  #Create complex number for the current pixel
            z = 0j

            escape_iter = max_iter #Initialize escape iteration count to max_iter (assume it does not escape)

            for n in range(max_iter):
                z = z * z + c #Iterate the Mandelbrot function
                if abs(z) > ESCAPE_RADIUS:#Check if the magnitude of z exceeds the escape radius
                    escape_iter = n + 1
                    break

            result[row, col] = escape_iter / max_iter #Normalize escape iteration count to [0, 1]

    return result
```

This version is the most straightforward and easiest to understand.
However, every arithmetic operation is executed through the Python interpreter, which introduces significant overhead.

As a result, this approach becomes very slow for large images.

### 2.2 NumPy Vectorized Implementation
The NumPy implementation stores the full complex grid in arrays and performs operations on many pixels simultaneously.  

```python
def mandelbrot_numpy(width, height, max_iter):
    x = np.linspace(X_MIN, X_MAX, width)  # Create evenly spaced real-axis values
    y = np.linspace(Y_MIN, Y_MAX, height)  # Create evenly spaced imaginary-axis values
    X, Y = np.meshgrid(x, y)  # Build the full 2D grid of coordinates
    C = X + 1j * Y  # Create the complex grid C = x + iy for all pixels at once

    Z = np.zeros_like(C, dtype=np.complex128)  # Initialize Z to 0 for all points
    result = np.ones(C.shape, dtype=np.float64)  # Initialize result array; default 1.0 means "did not escape"
    active = np.ones(C.shape, dtype=bool)  # Boolean mask to track points that are still active

    for n in range(max_iter):
        Z[active] = Z[active] * Z[active] + C[active]  # Update only points that have not escaped yet

        escaped_now = np.abs(Z) > ESCAPE_RADIUS  # Check which points have exceeded the escape radius
        newly_escaped = escaped_now & active  # Keep only the points that escaped in this iteration

        result[newly_escaped] = (n + 1) / max_iter  # Store normalized escape iteration for newly escaped points
        active[newly_escaped] = False  # Mark newly escaped points as inactive

        if not np.any(active):
            break  # Stop early if all points have already escaped

    result[active] = 1.0  # Points still active after max_iter are considered inside the Mandelbrot set
    return result

```
Instead of iterating pixel by pixel, NumPy executes operations on entire arrays using optimized C code.

A boolean mask tracks which points have not yet escaped. Once a point escapes, it is removed from further computation.

This significantly reduces Python overhead and improves performance.

### 2.3 Numba Implementation (JIT Compilation)
The third approach uses Numba, and compiles the Python function into native machine code the first time it is called.

The algorithm remains almost identical to the naive version, but after compilation it runs at near C-level speed.

A warm-up call is performed before benchmarking to trigger compilation so that the measured runtime only reflects execution time.
```python 
@njit
def mandelbrot_numba(width, height, max_iter, x_min, x_max, y_min, y_max, escape_radius):
    result = np.zeros((height, width), dtype=np.float64)  # Create a 2D array to hold the escape values

    for row in range(height):
        for col in range(width):
            x = x_min + (col / (width - 1)) * (x_max - x_min)  # Map pixel column to real part
            y = y_min + (row / (height - 1)) * (y_max - y_min)  # Map pixel row to imaginary part

            cr = x  # Real part of the complex number c
            ci = y  # Imaginary part of the complex number c
            zr = 0.0  # Real part of z, initialized to 0
            zi = 0.0  # Imaginary part of z, initialized to 0

            escape_iter = max_iter  # Initialize escape iteration count to max_iter (assume it does not escape)

            for n in range(max_iter):
                zr_new = zr * zr - zi * zi + cr  # Compute new real part of z^2 + c
                zi_new = 2.0 * zr * zi + ci  # Compute new imaginary part of z^2 + c
                zr = zr_new  # Update real part of z
                zi = zi_new  # Update imaginary part of z

                if zr * zr + zi * zi > escape_radius * escape_radius:  # Check escape condition without square root
                    escape_iter = n + 1
                    break

            result[row, col] = escape_iter / max_iter  # Normalize escape iteration count to [0, 1]

    return result
``` 
### 3. Benchmarking Methodology

To compare the implementations, a benchmarking procedure was performed.

Key aspects of the methodology:

Grid sizes tested: several square grids (NxN) with increasing resolution.

Timing method: time.perf_counter() was used to measure wall-clock execution time.

Multiple repetitions: each configuration was executed several times and the mean runtime was computed.

Numba warm-up: a small test run was executed before benchmarking to exclude compilation time.

The recorded timing data was exported to a CSV file: results/timings.csv

This file contains the execution time measurements for all experiments.
### 4. Experimental Results

The performance results show clear differences between the three implementations.

### General trends

The naive implementation is the slowest.

The NumPy implementation significantly reduces runtime by using vectorized operations.

The Numba implementation typically provides the best performance due to compiled execution.

The benchmark plots generated in the notebook show the relationship between problem size (number of pixels) and execution time.

As expected, execution time increases as the grid resolution increases.

### 5. Interpretation
### Naive Implementation
The naive implementation is slow because Python executes each arithmetic operation through its interpreter. Each iteration involves dynamic type handling and function calls, which introduces large overhead.

For large grids, the number of operations grows rapidly, making this method impractical for high-resolution images.
### NumPy Vectorization
NumPy improves performance by moving most computations into optimized C routines.

Instead of iterating pixel by pixel in Python, operations are performed on entire arrays at once. This greatly reduces interpreter overhead.

However, the vectorized implementation still performs the outer iteration loop in Python, which limits its performance. 
### Numba Compilation
Numba provides the highest performance by compiling the loop-based algorithm to native machine code.

This allows the program to execute the nested loops efficiently without Python overhead.

Additionally, the loop can exit early for points that escape quickly, reducing unnecessary computations.
### Scaling Behaviour
All implementations scale approximately with the number of pixels in the grid.

Since the Mandelbrot set requires evaluating every pixel independently, the computational complexity grows with the size of the grid.

The difference between the implementations is mainly a constant performance factor, not a change in algorithmic complexity.

### 6. Conclusion
This project compared three methods for computing the Mandelbrot set in Python.
The results show that using compiled numerical methods can dramatically improve performance.

Numba offers the best performance among the tested methods while preserving the simple loop-based structure of the algorithm.

For large-scale Mandelbrot computations or similar numerical problems, JIT compilation or vectorized approaches are essential to achieve efficient performance.
